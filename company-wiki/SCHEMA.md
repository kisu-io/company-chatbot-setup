# Wiki Schema

## Domain
Company training materials, onboarding documentation, processes, and knowledge base.
Synced from Lark/Feishu Wiki.

## Conventions
- **File names:** lowercase, hyphens, no spaces (e.g., `onboarding-guide.md`)
- **Every wiki page starts with YAML frontmatter** (see below)
- **Use `[[wikilinks]]** to link between pages (minimum 2 outbound links per page)
- **When updating a page, always bump the `updated` date**
- **Every new page must be added to `index.md`**
- **Every action must be appended to `log.md`**
- **Provenance markers:** On pages that synthesize 3+ sources, append `^[raw/lark/source-file.md]` at the end of paragraphs whose claims come from a specific source.

## Frontmatter

### Wiki Pages (concepts, learning-paths, queries)
```yaml
---
title: Page Title
created: YYYY-MM-DD
updated: YYYY-MM-DD
type: entity | concept | comparison | query | summary | learning-path
tags: [from taxonomy below]
sources: [raw/lark/source-name.md]
confidence: high | medium | low
contested: true  # optional, when contradictions exist
contradictions: [other-page-slug]  # optional
---
```

### Raw Sync Files (auto-generated by sync script)
```yaml
---
source_url: https://feishu.cn/docs/xxx
ingested: YYYY-MM-DD
sha256: <hex digest>
lark_title: Original Title
lark_id: document_id
---
```

## Tag Taxonomy

### Content Type
- `onboarding` — New hire materials
- `product` — Product documentation
- `engineering` — Technical docs, code standards
- `sales` — Sales processes, scripts
- `marketing` — Marketing materials
- `hr` — HR policies, benefits
- `compliance` — Legal, compliance docs
- `operations` — Operational procedures

### Difficulty Level
- `beginner` — Entry level
- `intermediate` — Some experience needed
- `advanced` — Expert level

### Requirement Status
- `required` — Must read/do
- `optional` — Nice to know
- `reference` — For lookup only

### Meta Tags
- `comparison` — Side-by-side analysis
- `timeline` — Chronological info
- `checklist` — Action items
- `faq` — Frequently asked questions
- `template` — Reusable templates

**Rule:** Every tag on a page must appear in this taxonomy. If a new tag is needed, add it here first, then use it.

## Page Thresholds

- **Create a concept page** when a topic appears in 2+ source docs OR is central to one important doc
- **Add to existing page** when a source mentions something already covered
- **DON'T create a page** for passing mentions, minor details, or things outside the domain
- **Split a page** when it exceeds ~200 lines — break into sub-topics with cross-links
- **Archive a page** when its content is fully superseded — move to `_archive/`, remove from index

## Page Types

### Entity Pages (`entities/`)
One page per notable entity (person, team, product, tool).
Include:
- Overview / what it is
- Key facts and dates
- Relationships to other entities (`[[wikilinks]]`)
- Source references

### Concept Pages (`concepts/`)
One page per concept or topic.
Include:
- Definition / explanation
- Current state of knowledge
- Open questions or debates
- Related concepts (`[[wikilinks]]`)

### Learning Paths (`learning-paths/`)
Curated sequences for specific goals.
Include:
- Target audience
- Prerequisites
- Ordered list of docs/concepts to study
- Estimated time
- Success criteria

Example:
```markdown
# Learning Path: New Engineer Onboarding

**Audience:** New hire engineers
**Prerequisites:** None
**Duration:** 2 weeks

## Week 1: Foundations
1. [[company-mission]]
2. [[tech-stack-overview]]
3. [[dev-environment-setup]]

## Week 2: Deep Dive
1. [[code-review-process]]
2. [[deployment-pipeline]]
3. [[incident-response]]

## Verification
- [ ] Complete dev setup
- [ ] Submit first PR
- [ ] Shadow on-call engineer
```

### Query Pages (`queries/`)
Filed Q&A that's worth keeping.
Include:
- The question
- Synthesized answer
- Source documents
- Related queries

## Update Policy

When new information conflicts with existing content:

1. **Check the dates** — newer sources generally supersede older ones
2. **If genuinely contradictory**, note both positions with dates and sources
3. **Mark the contradiction** in frontmatter: `contradictions: [page-name]`
4. **Flag for user review** in the lint report

## Index Structure

`index.md` should be sectioned:

```markdown
# Company Wiki Index

> Last updated: YYYY-MM-DD | Total pages: N

## Raw Sources (from Lark)
<!-- Auto-generated by sync script -->

## Concepts
<!-- Manually curated concept pages -->

## Learning Paths
<!-- Curated learning sequences -->

## Queries
<!-- Filed Q&A -->
```

## Log Format

`log.md` entries:

```markdown
## [YYYY-MM-DD] action | subject

Details of what changed...

Files modified:
- path/to/file1.md
- path/to/file2.md
```

Actions: `sync`, `create`, `update`, `query`, `lint`, `archive`

## Quality Gates

Before a page is considered "complete":

- [ ] Has required frontmatter fields
- [ ] Has at least 2 outbound `[[wikilinks]]`
- [ ] Appears in `index.md`
- [ ] Tags are from taxonomy
- [ ] Sources are cited
- [ ] Confidence level is set (if opinion-heavy)
- [ ] Added to `log.md`
