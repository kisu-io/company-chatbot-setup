📋 Overview

This guide walks you through setting up an AI-powered learning chatbot that syncs your company's Lark Wiki documentation and makes it searchable via natural language queries.

⏱️ Time Required

90-120 minutes for complete setup

🎯 Difficulty

Intermediate (requires API setup + Python)

📦 What You'll Build

Automated wiki sync + Q&A chatbot on Telegram

Architecture: Lark Wiki → Sync Script → Local Markdown → Hermes Agent → Telegram Users

✅ Pre-Session Checklist

Complete these tonight before tomorrow's setup session:

Security: Never commit API credentials to git. Use environment variables or secret management.

🛠️ Setup Steps

Phase 1: Install Hermes (15 min)

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
hermes --version
python3 --version  # Need 3.10+

Phase 2: Copy Config Files (10 min)

cd ~/projects/company-chatbot
cp config.yaml.example ~/.hermes/config.yaml
cp .env.example ~/.hermes/.env
nano ~/.hermes/.env  # Fill in Lark credentials

Phase 3: Install Dependencies (10 min)

pip3 install requests python-dotenv markdown
python3 -c "import requests, dotenv; print('OK')"

Phase 4: Test Sync Script (20 min)

chmod +x scripts/sync-lark-wiki.py
python3 scripts/sync-lark-wiki.py

Expected: All docs synced to company-wiki/raw/lark/

Phase 5: Test Hermes Queries (15 min)

hermes
# Ask: "What documents do we have about onboarding?"

Phase 6: Deploy to Telegram (15 min)

hermes gateway start
# Test from Telegram: "What's in our company wiki?"

Phase 7: Auto-Sync Cron (15 min, optional)

crontab -e
# Add: 0 */6 * * * python3 ~/projects/company-chatbot/scripts/sync-lark-wiki.py

Key Files Explained

sync-lark-wiki.py

Main sync script. Fetches docs from Lark API, converts to markdown, detects changes via SHA256, updates index.md and log.md automatically.

SCHEMA.md

Wiki conventions: tag taxonomy, frontmatter format, page types, quality gates. Based on Karpathy's LLM Wiki pattern.

config.yaml

Hermes Agent configuration: models, providers, MCP servers (optional), gateway settings, memory backend.

.env

Environment variables: Lark credentials, wiki path, model overrides. Never commit this file.

🎯 Implementation Approaches

Approach Best For Pros Cons
Local Sync <100 docs, weekly updates Simple, fast queries, no rate limits Not real-time, needs sync script
MCP Bridge 500+ docs, daily updates Always fresh, no sync complexity Needs MCP server, API rate limits
Hybrid 100-500 docs, weekly updates Fast cache + fresh fallback More complex setup
Recommendation: Start with Local Sync. You can add live MCP queries later if needed.

🐛 Troubleshooting

❌ Auth Failed

Check LARK_APP_ID and LARK_APP_SECRET in ~/.hermes/.env. Verify in Lark developer console.

❌ Permission Denied

Enable Docs:Read permission in Lark app → Permissions → Enable.

❌ Rate Limited

Increase sleep delays in sync script: time.sleep(1) → time.sleep(3)

❌ Wiki Not Found

Set WIKI_PATH in ~/.hermes/.env and restart Hermes.
